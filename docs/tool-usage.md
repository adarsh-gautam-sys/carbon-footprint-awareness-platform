# Tool Usage Explanation

## Google Tools

- Gemini API: Generates concise, personalized carbon reduction insights from validated footprint data.
- Cloud Run: Hosts the FastAPI app as a containerized web service.
- Cloud Build: Used by `gcloud run deploy --source` to build the service.
- Artifact Registry: Stores build artifacts generated by Cloud Run source deployment.
- Secret Manager: Recommended for `GEMINI_API_KEY` instead of committing keys.
- Firestore: Optional persistence layer for assessment history when `FIRESTORE_ENABLED=true`.

## MCP and Development Tools

- Sequential Thinking MCP: Configured in `mcp.json` for structured planning and review.
- FastAPI: Provides typed API endpoints and OpenAPI docs.
- Pydantic: Validates user input and prevents invalid carbon calculations.
- Pytest: Covers calculation, action generation, and Gemini fallback behavior.

## Why These Tools

The stack directly matches the PromptWars requirements: it uses Google AI for personalized reasoning, Google Cloud Run for deployment, structured backend logic for quality, and tests/docs for evaluation readiness.

## Environment Variables

- `GEMINI_API_KEY`: Gemini key loaded from Secret Manager or local `.env`.
- `GEMINI_MODEL`: Optional model override, default `gemini-2.0-flash`.
- `GOOGLE_CLOUD_PROJECT`: Google Cloud project id.
- `GOOGLE_CLOUD_REGION`: Deployment region.
- `FIRESTORE_ENABLED`: Set to `true` to store assessments in Firestore.
- `LOCAL_DATA_DIR`: Local JSONL storage path for development.
